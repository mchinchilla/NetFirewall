# NetFirewall appliance ISO (Debian 13 / trixie, live-build)

Builds a hybrid ISO that is **both** a bootable live appliance (and VM image)
**and** a bare-metal installer. Everything is embedded so the install works
fully **offline** — the self-contained NetFirewall `.deb` plus every OS
dependency is pulled into the squashfs at build time.

## Layout

```
deploy/iso/
  auto/{config,build,clean}              lb wrappers (auto/config = the lb config knobs)
  config/
    package-lists/netfirewall.list.chroot  OS deps embedded in the squashfs
    archives/pgdg.*                        PGDG repo + key → PostgreSQL 18 (Debian 13 main only has 17)
    packages.chroot/                       CI drops netfirewall_*.deb here (gitignored)
    hooks/normal/0100-enable-services      enable postgresql + our units + nginx + ssh
    hooks/normal/0200-harden               hostname, lock root pw, sshd, ifupdown wiring
    includes.chroot/                       files copied verbatim into the image:
      etc/network/interfaces.d/00-netfirewall-bootstrap   DHCP-on-first-NIC (token __FIRST_NIC__)
      etc/network/if-{up,down}.d/netfirewall-banner       refresh /etc/issue on link change
      usr/local/sbin/netfirewall-nic-bootstrap            first-boot NIC substitution
      etc/systemd/system/netfirewall-nic-bootstrap.service + sysinit.target.wants symlink
      etc/issue                                           static console banner (pre-network)
    includes.installer/preseed.cfg         hands-off d-i: guided whole-disk, headless
    bootloaders/grub-pc/config.cfg         branded boot menu (live / install / failsafe)
```

## Build (Debian 13 host or container, as root)

```bash
# 1. Build the .deb first (see deploy/debian/) and drop it in:
cp ../../netfirewall_*.deb config/packages.chroot/

# 2. Install live-build + tooling:
apt-get install -y live-build debootstrap squashfs-tools xorriso \
    isolinux syslinux-common grub-pc-bin grub-efi-amd64-bin mtools dosfstools

# 3. Configure + build:
cd deploy/iso
lb config        # reads auto/config
lb build         # ~20-40 min depending on mirror speed + disk; produces live-image-amd64.hybrid.iso
```

`lb build` needs root and loop/mount/debootstrap — run in a privileged
container or directly on a Debian host. In CI we use `container: debian:trixie`
with `options: --privileged` (see `.github/workflows/build-iso.yml`).

## First boot (live or installed)

1. `netfirewall-nic-bootstrap` substitutes the real first NIC into the bootstrap
   interfaces file → DHCP → the appliance is reachable.
2. `netfirewall-firstboot.service` provisions per-appliance secrets + the local
   PostgreSQL DB, runs migrations + seeds, generates a self-signed TLS cert at
   `/etc/netfirewall/tls/` and enables the nginx reverse proxy (HTTPS from first
   boot), starts the daemon/web (see the .deb's README.Debian). Drops
   `/var/lib/netfirewall/.provisioned` on success.
3. The console `/etc/issue` shows the acquired IP + the **https://** web URL.
   First login uses the bootstrap token: `journalctl -u netfirewall-web | grep -i token`.
   The cert is self-signed → expect a browser warning until you install a real one.

## Test in QEMU

```bash
qemu-system-x86_64 -m 4096 -smp 2 -enable-kvm \
    -netdev user,id=n0 -device virtio-net,netdev=n0 \
    -cdrom deploy/iso/live-image-amd64.hybrid.iso
```

Add a second NIC to exercise WAN/LAN role assignment in the SetupWizard.

## The #1 rule: no secrets in the image

The squashfs `/etc/netfirewall/*.env` MUST still contain the `__REPLACE__` /
`__REPLACE_MASTER_KEY__` placeholders — the per-appliance master key + DB
password are generated by firstboot, never baked in. CI asserts this by mounting
the built squashfs and grepping the env files; the build fails if a real base64
key leaked in. See `docs/master-key.md`.
